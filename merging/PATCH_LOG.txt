### 🔧 Fixed Bug
**File**: `Tokenizer.cpp` (Line 18)

**Issue**: Size_t underflow in reverse loop
```cpp
// BEFORE (BROKEN):
for(size_t i = size - 1; i >= 0; i--)  // i >= 0 always true for size_t, underflows

// AFTER (FIXED):
for(int i = (int)size - 1; i >= 0; i--)  // Proper signed comparison
{
    if(s[i] != ' ' && s[i] != '\t' && s[i] != '\n' && s[i] != '\v' && s[i] != '\f' && s[i] != '\r')
    {
        if((size_t)i == size - 1)
            break;
        s = s.substr(0, i + 1);
        break;
    }
}
```

**Note**: Pre-existing bug (author's code, not introduced by merge). Would fail -Werror compilation.

```cpp
// AFTER:

```

**Change 2**: Stub tryCGI function
```cpp
// BEFORE:
static bool tryCGI(const HTTPRequest &request, const RouteConfig &route, Response &resp) {
  CGIHandler cgiHandler;
  CGIResult cgiResult;
  if (!cgiHandler.isCGIRequest(request.getURI(), route))
    return false;
  // ... full CGI logic
}

// AFTER:
static bool tryCGI(const HTTPRequest &request, const RouteConfig &route, Response &resp) {
#ifdef HAVE_CGI
  CGIHandler cgiHandler;
  CGIResult cgiResult;
  if (!cgiHandler.isCGIRequest(request.getURI(), route))
    return false;
  // ... full CGI logic
  return true;
#else
  // CGI stub: Person 1 will enable this with -DHAVE_CGI
  (void)request;
  (void)route;
  (void)resp;
  return false;
#endif
}
```

**Reason**: Person 1 hasn't implemented CGIHandler yet. Stub allows compilation without CGI, ready to flip on with `-DHAVE_CGI` flag.

## RouteConfig (NEW FILE)

**What it is**: Adapter that presents a `(server_block, location_block*)` pair as the interface MethodHandler expects.

**Key Transformations**:
1. **Methods**: `map<string,bool>` → `vector<string>` (flattens, only true keys)
2. **Redirect**: `map<int,string>` → `(code, url)` tuple (takes first entry; parser guarantees ≤1)
3. **Priority**: Location fields override server fields (e.g., root, index)

**Files**:
- `includes/http/RouteConfig.hpp` — Interface (NEW)
