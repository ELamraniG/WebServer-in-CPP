Why node_02

What this instance does.

A second webserv process with its own config file, port, and route table. Different behavior, same binary.

Autoindex ON
Every directory request returns a styled listing rendered from the autoindex.html template. Files are sorted, typed, and linkable.
File Upload
POST multipart/form-data directly to /uploads. Boundary parsed in C++, saved to disk. No CGI needed.
DELETE Enabled
Files in /uploads/ can be removed via HTTP DELETE. Returns 204 on success — no body, no noise.
301 Redirects
Config-defined redirects via the return directive. No CGI, no handlers — pure config-level Location header.
Separate Config
Runs from its own .conf file — different root, different error pages, different max body size. Same binary, zero code changes.
Timeout Handling
CGI processes that exceed the timeout threshold are killed → 504. Slow clients that stall mid-request → 408.
