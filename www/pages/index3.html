HTTP Methods

What webserv handles.

Every method implemented from scratch — no libc HTTP helpers, no frameworks. Just raw parsing and POSIX I/O.

GET
Retrieve a resource
Serves static files, triggers CGI, or returns a directory listing when autoindex is on. The workhorse of the server.
200 OK 404 Not Found 403 Forbidden
POST
Submit data
Handles multipart/form-data file uploads and passes body data to CGI scripts via CONTENT_LENGTH + stdin pipe.
201 Created 200 OK 413 Too Large
DELETE
Remove a resource
Deletes files from /uploads/ when the route allows it. Returns 204 on success, 403 or 404 otherwise.
204 No Content 404 Not Found 403 Forbidden
HEAD
Headers only
Identical to GET but the response body is suppressed. Useful for checking if a resource exists without downloading it.
200 OK 404 Not Found

Request lifecycle

How a request flows.

Every HTTP request passes through three layers — each owned by a different person on the team.

REDA · Network
TCP + poll()
  • accept() connection
  • poll() readiness
  • recv() raw bytes
  • CGI Processing
→
ELAMRANI · Protocol
HTTP Parsing
  • Parse request line
  • Parse headers + body
  • Handle methods
  • Build HTTP response
→
SIMO · Application
Route + Serve
  • Match location config
  • Serve file or error
  • Directory autoindex
  • POLLOUT → send()
